    Анализ документации Apps Service Opus

    Дата анализа: 2026-04-16 Аналитик: AI-ассистент

    Резюме

    Документация обширная, но противоречивая. Многое описывает функциональность, которая не реализована или работает не так, как задумано. Критическая проблема: примеры кода содержат ошибки, которые приведут к неработающей конфигурации.

    1. Чего не хватает (пропущенные разделы)

    1.1 Отсутствующие гайды для разработчиков

    Из docs/development/guides-wishlist.md — следующие гайды не написаны:

    Гайд Критичность Почему нужен
    Локальная разработка — первый запуск 🔴 Высокая Новый разработчик не знает как запустить проект
    Git workflow — ветки, коммиты, PR 🟠 Средняя Непонятно как работать с ветками
    Добавление нового API endpoint 🔴 Высокая Нет руководства по расширению API
    NiceGUI — как добавлять страницы 🟠 Средняя UI разрабатывается вслепую
    Аутентификация — Keycloak vs Builtin 🔴 Высокая Критичный компонент, нет документации
    Caddy шаблоны — как редактировать 🟠 Средняя Любой баг в routing требует правки шаблонов
    Platform CLI — добавление команды 🟡 Низкая Для расширения CLI
    Troubleshooting / FAQ 🔴 Высокая Нет централизованного решения проблем

    1.2 Отсутствующие разделы в пользовательской документации

    • Полный reference service.yml — нет документации всех полей манифеста
    • Устранение неполадок (Troubleshooting) — нет раздела "если не работает..."
    • Миграция сервисов — как перенести сервис с одного сервера на другой
    • Удаление сервисов — как правильно удалить сервис и все его данные
    • Настройка Keycloak — подробная инструкция по интеграции
    • Настройка Telegram бота — пошаговая инструкция
    • Работа с секретами — где и как хранить DATABASE_URL, API_KEY
    • Ограничения ресурсов — как настраивать memory/cpu limits в Docker

    1.3 Отсутствующая архитектурная документация

    • ER-диаграмма базы данных — нет схемы SQLite
    • Sequence diagrams — нет диаграмм последовательности для ключевых сценариев
    • Описание EventBus — как работает pub/sub между компонентами
    • File watcher — как работает отслеживание изменений

    2. Что плохо написано (качество документации)

    2.1 Критические ошибки в примерах кода

    ❌ Проблема: container_name отсутствует в примерах

    Файлы: - docs/examples.md — 6 примеров, только часть имеет container_name - docs/getting-started/first-service.md — минимальный пример без container_name - docs/user-guide/services.md — примеры без container_name - docs/architecture.md — пример манифеста без container_name

    Почему это критично: Без container_name Caddy использует host.docker.internal:<port> — проксирование на хост-машину вместо контейнера. Это: - Обходит SSL и аутентификацию Caddy - Создаёт конфликты портов - Является дыркой в безопасности

    Исправление: ВСЕ примеры должны содержать container_name, совпадающий с именем контейнера в docker-compose.yml.

    ❌ Проблема: docker-compose.yml без container_name

    Примеры в документации часто не содержат container_name в docker-compose.yml, но используют его в service.yml — это несоответствие.

    2.2 Противоречия между разделами

    Раздел 1 Утверждение 1 Раздел 2 Утверждение 2
    docs/user-guide/monitoring.md Loki интегрирован docs/index.md Loki — не реализован
    docs/user-guide/backup.md Restic интегрирован docs/index.md Restic upload — скрипты не написаны
    docs/examples.md type: static поддерживается docs/index.md Только type: docker-compose реализован
    docs/api.md Все эндпоинты требуют Bearer токен docs/api.md Нет login endpoint для получения токена

    2.3 Непонятные объяснения

    "Local override" (service.local.yml)

    • Нет чёткого объяснения приоритета: что перезаписывает что
    • Нет примера "до и после" merge
    • Не объяснено, что происходит при конфликтах массивов

    "Routing types"

    • Нет схемы/диаграммы показывающей разницу между domain/subfolder/port
    • Не объяснено, когда что использовать
    • Нет примеров комбинирования типов

    "Health checks"

    • Не объяснено, что происходит при failed health check
    • Не описан алгоритм retries и backoff
    • Нет информации о таймаутах в коде (хотя в манифесте они есть)

    2.4 Избыточность и дублирование

    Информация Где дублируется
    Установка README.md, docs/getting-started/install.md
    Быстрый старт README.md, docs/getting-started/first-service.md
    Статус компонентов README.md, docs/index.md
    Обновление платформы docs/development/deployment-runbook.md, docs/development/upgrade-to-release.md, docs/plan/deployment-runbook.md

    2.5 Неполные инструкции

    Установка Platform CLI

    • В docs/getting-started/cli-ui.md — просто "Python CLI на Typer"
    • Нет инструкции по установке pipx
    • Нет инструкции по созданию venv
    • Нет проверки что CLI работает

    Настройка Keycloak

    • Упоминается в нескольких местах
    • Нет пошаговой инструкции
    • Нет примера конфигурации realm/client

    3. Добавить предупреждения о неработающем функционале — Loki, Restic, static type
    4. Создать troubleshooting раздел — с решениями типовых проблем

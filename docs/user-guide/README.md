# Руководство пользователя по развертыванию и настройке платформы

Это руководство поможет вам развернуть и настроить платформу apps-service-opus на вашем сервере Ubuntu.

## 1. Системные требования

Для работы платформы необходимы следующие компоненты:

- Ubuntu 20.04 или выше
- Docker 20.10 или выше
- Docker Compose 1.29 или выше
- Не менее 4 ГБ оперативной памяти
- Не менее 20 ГБ свободного места на диске
- Доступ к интернету для загрузки образов

## 2. Пошаговое руководство по установке и настройке платформы

### 2.1. Подготовка системы

1. Обновите систему:
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

2. Установите Docker и Docker Compose:
   ```bash
   sudo apt install docker.io docker-compose -y
   ```

3. Добавьте текущего пользователя в группу docker:
   ```bash
   sudo usermod -aG docker $USER
   ```
   
   Выйдите из системы и войдите снова для применения изменений.

### 2.2. Клонирование репозитория

1. Перейдите в директорию `/opt`:
   ```bash
   cd /opt
   ```

2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-organization/apps-service-opus.git
   ```

3. Перейдите в директорию проекта:
   ```bash
   cd apps-service-opus
   ```

### 2.3. Настройка переменных окружения

1. Создайте файл `.env` в корневой директории проекта:
   ```bash
   cp .env.example .env
   ```

2. Откройте файл `.env` и настройте переменные окружения:
   ```bash
   nano .env
   ```

   Основные параметры для настройки:
   - `TELEGRAM_BOT_TOKEN` - токен Telegram бота для уведомлений
   - `KEYCLOAK_ADMIN_PASSWORD` - пароль администратора Keycloak
   - `DOMAIN` - доменное имя вашего сервера

## 3. Описание процесса инициализации платформы

Платформа состоит из нескольких основных компонентов:

- **Master Service** - центральный сервис управления
- **Caddy** - reverse proxy и веб-сервер
- **Keycloak** - сервис аутентификации и авторизации
- **Сервисы приложений** - пользовательские сервисы

При инициализации платформа автоматически создает необходимую структуру директорий и конфигурационные файлы.

## 4. Инструкции по запуску Master Service, Caddy и других компонентов

### 4.1. Запуск Master Service

1. Перейдите в директорию Master Service:
   ```bash
   cd apps/_core/master
   ```

2. Запустите сервис с помощью Docker Compose:
   ```bash
   docker-compose up -d
   ```

### 4.2. Запуск Caddy

1. Перейдите в директорию Caddy:
   ```bash
   cd ../../_core/caddy
   ```

2. Запустите Caddy:
   ```bash
   docker-compose up -d
   ```

### 4.3. Запуск Keycloak

Keycloak запускается автоматически вместе с Master Service при выполнении команды:
```bash
docker-compose up -d
```

в директории `apps/_core/master`.

## 5. Настройка переменных окружения

Основные переменные окружения находятся в файле `.env` в корневой директории проекта:

- `DOMAIN` - основной домен платформы
- `TELEGRAM_BOT_TOKEN` - токен Telegram бота для уведомлений
- `KEYCLOAK_ADMIN_PASSWORD` - пароль администратора Keycloak
- `DATABASE_URL` - URL базы данных (по умолчанию используется SQLite)

Дополнительные переменные можно найти в файлах `docker-compose.yml` каждого сервиса.

## 6. Конфигурация Telegram уведомлений

1. Создайте Telegram бота через @BotFather и получите токен.

2. Добавьте токен в файл `.env`:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен
   ```

3. Получите ваш Telegram ID через @userinfobot.

4. Добавьте ваш ID в файл `.env`:
   ```
   TELEGRAM_CHAT_IDS=["ваш_telegram_id"]
   ```

5. Перезапустите Master Service:
   ```bash
   cd /opt/apps-service-opus/apps/_core/master
   docker-compose restart
   ```

## 7. Настройка Keycloak для аутентификации

### 7.1. Первичная настройка

1. После запуска Keycloak будет доступен по адресу:
   ```
   http://ваш_домен:8080
   ```

2. Войдите в админ-панель:
   - Логин: admin
   - Пароль: значение переменной `KEYCLOAK_ADMIN_PASSWORD` из файла `.env`

### 7.2. Создание realm и клиента

1. В админ-панели создайте новый realm с именем `platform`.

2. В созданном realm создайте клиента:
   - Client ID: `master-service`
   - Client Protocol: `openid-connect`
   - Root URL: оставьте пустым

3. Настройте клиент:
   - Access Type: `confidential`
   - Service Accounts Enabled: `ON`
   - Authorization Enabled: `ON`

4. Сохраните настройки и перейдите на вкладку Credentials для получения секрета клиента.

5. Добавьте секрет клиента в файл `.env` в переменную `KEYCLOAK_CLIENT_SECRET`.

### 7.3. Создание ролей и пользователей

1. В realm `platform` создайте роли:
   - `admin` - для администраторов платформы
   - `user` - для обычных пользователей

2. Создайте пользователей и назначьте им соответствующие роли.

## 8. Проверка работоспособности после установки

### 8.1. Проверка запущенных контейнеров

Выполните команду:
```bash
docker ps
```

Убедитесь, что запущены следующие контейнеры:
- platform-master
- caddy
- keycloak

### 8.2. Проверка доступности сервисов

1. Master Service должен быть доступен по адресу:
   ```
   http://ваш_домен:8000
   ```

2. Keycloak должен быть доступен по адресу:
   ```
   http://ваш_домен:8080
   ```

### 8.3. Проверка логов

Для проверки логов сервисов используйте команду:
```bash
docker logs имя_контейнера
```

Например:
```bash
docker logs platform-master
```

### 8.4. Проверка уведомлений

После настройки Telegram уведомлений, при запуске платформы вы должны получить сообщение в Telegram о старте сервиса.

## 4. Проблемы и решения

### 4.1 Конфликт poetry.lock
**Проблема:** При попытке запуска `poetry install` в контейнере возникла ошибка из-за несовместимости версий зависимостей в `poetry.lock`.

**Решение:** 
- Вместо использования `poetry install` внутри контейнера применен подход с предварительной сборкой зависимостей через Docker multi-stage build.
- Обновлен `Dockerfile` master-сервиса для копирования `poetry.lock` и `pyproject.toml` с последующей установкой через `poetry install --no-dev`.
- Для ускорения процесса использован кэш слоев Docker.

### 4.2 Ошибки NiceGUI в логах
**Проблема:** В логах master-сервиса наблюдались повторяющиеся предупреждения:
```
WARNING: nicegui: Could not establish WebSocket connection
```
и
```
ERROR: nicegui: Event loop already running
```

**Решение:**
- Проверена конфигурация WebSocket в Caddy (проксирование `/nicegui`).
- Обновлена версия NiceGUI до `3.7.1` в `pyproject.toml`.
- В код добавлена явная инициализация NiceGUI с параметром `reconnect_timeout=10`.
- Ошибка "Event loop already running" была вызвана конфликтом асинхронных циклов; исправлено путем использования `asyncio.get_event_loop()` вместо создания нового.

### 4.3 Задержка запуска сервисов
**Проблема:** После перезапуска Caddy некоторые внутренние сервисы не были сразу доступны, что привело к ошибкам 502.

**Решение:**
- Добавлена пауза в 30 секунд после запуска Caddy для инициализации прокси.
- В скрипт `restart_core.sh` добавлена проверка готовности Caddy через `curl --retry 5`.
- Настроены health checks для внутренних сервисов в `service.yml`.

### 4.4 Проблема с platform-cli
**Проблема:** Команда `ops list` (из platform-cli) не работала после обновления, так как platform-cli не был установлен в изолированном venv.

**Решение:**
- Установлен platform-cli в изолированное окружение согласно инструкции `_core/platform-cli/INSTALL.md`.
- Обновлены переменные окружения для корректного пути к CLI.
- В качестве временного решения использован fallback на `docker ps` для отображения сервисов.

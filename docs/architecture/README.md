# Архитектура платформы управления сервисами

## 1. Общее описание архитектуры платформы

Платформа управления сервисами представляет собой комплексное решение для автоматизации развертывания, управления и мониторинга различных типов сервисов на сервере Ubuntu. Архитектура платформы построена по принципу микросервисов и включает в себя следующие ключевые компоненты:

1. **Master Service** - центральный сервис управления, реализованный на Python с использованием FastAPI и NiceGUI
2. **Caddy Proxy** - обратный прокси-сервер для маршрутизации трафика
3. **Docker** - контейнеризация приложений
4. **Restic** - система резервного копирования

Архитектура платформы обеспечивает минимальное участие DevOps при развертывании и управлении сервисами, поддерживает различные типы сервисов (статические сайты, docker-контейнеры, docker-compose проекты) и обеспечивает централизованное управление бэкапами, обновлениями и логами.

## 2. Описание компонентов платформы

### Master Service

Master Service - это центральный компонент платформы, реализованный на Python с использованием FastAPI и NiceGUI. Он обеспечивает:

- Управление сервисами (развертывание, остановка, перезапуск)
- Мониторинг состояния сервисов
- Управление логами сервисов
- Управление бэкапами сервисов
- Генерацию конфигураций для Caddy
- Интеграцию с Keycloak для аутентификации
- Отправку уведомлений через Telegram

Master Service состоит из следующих модулей:

- API для программного взаимодействия
- Web UI на базе NiceGUI для визуального управления
- Модуль обнаружения сервисов
- Модуль управления Docker
- Модуль управления Caddy
- Модуль управления бэкапами
- Модуль проверки состояния сервисов
- Модуль управления логами
- Модуль отправки уведомлений

### Caddy Proxy

Caddy Proxy выступает в роли обратного прокси-сервера и обеспечивает:

- Маршрутизацию трафика к сервисам
- SSL/TLS терминацию
- Аутентификацию через Keycloak
- Лимитирование запросов
- Логирование запросов
- Защиту от атак

Caddy настроен на динамическую конфигурацию через API, что позволяет Master Service автоматически генерировать и применять конфигурации для новых сервисов.

### Docker

Docker используется для контейнеризации сервисов и обеспечивает:

- Изоляцию сервисов друг от друга
- Единообразное развертывание сервисов
- Управление зависимостями сервисов
- Масштабируемость сервисов

### Restic

Restic используется для резервного копирования данных сервисов и обеспечивает:

- Инкрементальные бэкапы
- Шифрование данных
- Дедупликацию данных
- Хранение бэкапов в различных хранилищах

         ▼                    ▼                    ▼

### Взаимодействие компонентов

1. Master Service сканирует директории сервисов для обнаружения новых или измененных сервисов
2. При обнаружении изменений Master Service генерирует конфигурации для Caddy
3. Master Service перезагружает конфигурацию Caddy через API
4. Caddy маршрутизирует входящий трафик к соответствующим сервисам
5. Master Service мониторит состояние сервисов через Docker API
6. Master Service управляет бэкапами сервисов через Restic
7. Master Service отправляет уведомления через Telegram
8. Пользователи аутентифицируются через Keycloak

## 4. Потоки данных между компонентами

1. **Поток управления**:

   - Пользователь взаимодействует с Master Service через Web UI или API
   - Master Service управляет сервисами через Docker API
   - Master Service управляет конфигурацией Caddy через Caddy API
   - Master Service управляет бэкапами через Restic

2. **Поток данных**:

   - Входящий трафик поступает в Caddy
   - Caddy маршрутизирует трафик к соответствующим сервисам
   - Сервисы обмениваются данными через внутреннюю сеть Docker
   - Логи сервисов собираются Master Service
   - Бэкапы сервисов создаются и управляются через Restic

3. **Поток уведомлений**:
   - Master Service отправляет уведомления через Telegram
   - Уведомления могут содержать информацию о состоянии сервисов, результатах бэкапов и т.д.

## 6. Используемые технологии и инструменты

### Основные технологии

1. **Python** - основной язык программирования для Master Service
2. **FastAPI** - фреймворк для создания API
3. **NiceGUI** - фреймворк для создания Web UI
4. **Docker** - контейнеризация приложений
5. **Caddy** - обратный прокси-сервер
6. **Restic** - система резервного копирования
7. **Keycloak** - система аутентификации и авторизации
8. **SQLite** - база данных для хранения состояния Master Service
9. **Jinja2** - шаблонизатор для генерации конфигураций

### Вспомогательные инструменты

1. **Docker Compose** - для оркестрации контейнеров
2. **Watchfiles** - для отслеживания изменений файлов
3. **Watchdog** - для отслеживания изменений файловой системы
4. **Croniter** - для работы с cron-выражениями
5. **Aiofiles** - для асинхронной работы с файлами
6. **Aiohttp** - для асинхронных HTTP-запросов
7. **PyYAML** - для работы с YAML-файлами
8. **SQLAlchemy** - ORM для работы с базой данных

## 7. Масштабируемость и отказоустойчивость

### Масштабируемость

1. **Горизонтальное масштабирование сервисов**:

   - Каждый сервис может быть масштабирован независимо через Docker Compose
   - Master Service поддерживает управление масштабированием сервисов

2. **Вертикальное масштабирование**:

   - Сервисы могут использовать ресурсы хоста в соответствии с их требованиями
   - Master Service позволяет ограничивать ресурсы сервисов

3. **Масштабирование Master Service**:
   - Master Service может быть масштабирован горизонтально при необходимости
   - Использует асинхронную архитектуру для обработки большого количества запросов

### Отказоустойчивость

1. **Отказоустойчивость сервисов**:

   - Docker обеспечивает автоматический перезапуск упавших контейнеров
   - Master Service мониторит состояние сервисов и может уведомлять о проблемах

2. **Отказоустойчивость Master Service**:

   - Master Service использует SQLite для хранения состояния
   - При сбое Master Service может быть перезапущен без потери данных

3. **Отказоустойчивость Caddy**:

   - Caddy автоматически перезагружает конфигурацию при изменениях
   - При сбое Caddy может быть перезапущен без потери конфигурации

4. **Отказоустойчивость бэкапов**:

   - Restic обеспечивает надежное хранение бэкапов
   - Master Service может автоматически восстанавливать сервисы из бэкапов

5. **Мониторинг и уведомления**:
   - Master Service мониторит состояние всех компонентов
   - При возникновении проблем отправляются уведомления через Telegram

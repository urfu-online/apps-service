## 2. Что плохо написано (качество документации)

### 2.1 Критические ошибки в примерах кода

#### ❌ Проблема: `container_name` отсутствует в примерах

**Файлы:**
- `docs/examples.md` — 6 примеров, только часть имеет `container_name`
- `docs/getting-started/first-service.md` — минимальный пример без `container_name`
- `docs/user-guide/services.md` — примеры без `container_name`
- `docs/architecture.md` — пример манифеста без `container_name`

**Почему это критично:**
Без `container_name` Caddy использует `host.docker.internal:<port>` — проксирование на хост-машину вместо контейнера. Это:
- Обходит SSL и аутентификацию Caddy
- Создаёт конфликты портов
- Является дыркой в безопасности

**Исправление:** ВСЕ примеры должны содержать `container_name`, совпадающий с именем контейнера в `docker-compose.yml`.

#### ❌ Проблема: `docker-compose.yml` без `container_name`

Примеры в документации часто не содержат `container_name` в `docker-compose.yml`, но используют его в `service.yml` — это несоответствие.

### 2.2 Противоречия между разделами

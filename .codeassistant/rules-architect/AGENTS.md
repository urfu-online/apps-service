# AGENTS.md – Правила для режима Architect

## Архитектурные принципы
- **Монорепозиторий с изоляцией**: `_core/master` (FastAPI + NiceGUI) и `_core/platform-cli` (CLI) разделены, но используют общие конфиги
- **Service discovery через файловую систему**: сканирование `services/{public,internal}/` каждые N секунд (watchdog)
- **Генерация конфигов Caddy из шаблонов**: Jinja2-шаблоны в `_core/caddy/templates/`; перезагрузка через API Caddy
- **Разделение public/internal**: internal сервисы не проксируются наружу, доступны только master service

## Ограничения системы
- **Health checks**: только HTTP GET, таймауты не настраиваются в коде (жестко закодированы)
- **Backup расписания**: используют croniter, но нет валидации синтаксиса cron
- **Отсутствие миграций БД**: используется SQLite (`master.db`) без Alembic; схема создается при старте
- **Логирование**: минимальное; логи Caddy могут быть отключены, логи приложения выводятся в stdout
- **Аутентификация**: поддержка Keycloak и built‑in auth, но переключение требует перезапуска

## Точки расширения
- **Добавление нового типа роутинга**: нужно редактировать шаблоны Caddy (`domain.caddy.j2`, `port.caddy.j2`, `subfolder.caddy.j2`)
- **Интеграция с внешним мониторингом**: health checks можно расширить через `HealthChecker.add_custom_check`
- **Поддержка новых бэкап‑стратегий**: наследование от `BackupStrategy` в `BackupManager`
- **Кастомные нотификации**: `Notifier` абстракция, можно добавить Telegram, Email, Slack

## Зависимости инфраструктуры
- **Caddy**: обязателен как reverse proxy; конфиги генерируются автоматически, но требуют перезагрузки через API
- **Docker**: обязателен для запуска сервисов; используется aiodocker для управления контейнерами
- **DinD (Docker‑in‑Docker)**: требуется для интеграционных тестов; должен быть доступен Docker socket
- **Platform CLI**: изолированный venv, не должен конфликтовать с глобальными пакетами

## Рекомендации по проектированию новых сервисов
- **Манифест `service.yml`**: должен содержать `health_check.endpoint`, иначе health check не будет работать
- **Роутинг**: выбрать один из типов: `domain`, `subfolder`, `port`; комбинирование не поддерживается
- **Бэкапы**: указать `backup` секцию с `type` (`rsync` или `pg_dump`) и `schedule` (cron‑выражение)
- **Локальная разработка**: использовать `service.local.yml` для переопределения настроек (не коммитится)
